# The Studio Handbook

The **distilled, evergreen** playbook — the rules that survived contact with
reality, in a few tight pages. `game-context/` and `app-context/` remain the
*living research* (how we got here); this is *what we now believe*. When they
conflict, update the handbook. Every page is a ≤2-minute read.

| Page | The one thing |
|---|---|
| [00-strategy](00-strategy.md) | Classic mechanic + exactly **one** innovation; hold a hard complexity budget. |
| [01-apple-rules](01-apple-rules.md) | Genre mechanics are safe; never clone a specific hit. Submission gotchas. |
| [02-validation](02-validation.md) | The gates + thresholds: greybox → fun gate → ★distribution gate → TestFlight D1≥35% → soft-launch D7≥12%. |
| [**03-distribution** ★](03-distribution.md) | **The weak spot, codified.** Building isn't the bottleneck — distribution is. The daily/share/k-loop discipline. **Read this one most.** |
| [04-prior-art-audit](04-prior-art-audit.md) | The repeatable differentiation audit (App Store / Steam / USPTO / web). |
| [05-kit-template/](05-kit-template/) | The project kit blanks: CONTEXT / PRD / PRIOR-ART / PROTOTYPE-PLAN. |
| [06-lessons](06-lessons.md) | The graveyard — one line each, why it died. **The most valuable page.** |

The code half of the studio (`packages/`, `templates/`) exists to *serve* page 03:
`@jfun/growth-loop` makes "ship with a loop" the default so the lesson on page
06 stops repeating.
